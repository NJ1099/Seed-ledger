프라이버시 안내

이 사이트는 입력하신 자산 데이터를 서버에 저장하지 않습니다. 데이터가 어디에 어떻게 흐르는지 그대로 적었습니다.

① 데이터가 저장되는 곳
  • 입력하신 자산·거래·스냅샷·개인 이벤트 → 이 브라우저의 localStorage (이 기기에만, 서버 불가시).
  • 시세 캐시 (티커별 현재가, 환율 등) → 서버 공유 캐시. 모든 사용자가 동일한 캐시를 사용합니다. 누가 어떤 종목을 보유 중인지는 서버가 알 수 없습니다 (조회 시 티커 심볼만 전달).
  • 경제 캘린더 Top 5 → 서버 (공유 콘텐츠, 토스증권 공개 API 에서 6시간마다 자동 수집).
② 서버가 보는 것 / 못 보는 것

볼 수 있음:

  • 시세 조회한 티커 심볼 (예: AAPL, KRW-BTC, 005930) — 익명, 어떤 사용자가 조회했는지 매핑하지 않습니다.
  • IP 주소 (HTTP 요청 표준 메타데이터). 로그에 남을 수 있으나 자산 정보와 연결되지 않습니다.

볼 수 없음:

  • 보유 수량, 잔액, 평균단가, 평가금액, 손익.
  • 거래 내역 (카드 사용, 월세, 저축 등 모든 항목).
  • 계좌 라벨, 메모, 만기일 등 모든 사용자 입력 내용.
③ 텔레그램 동기화 시 격리 보장
  • 저장소 = 본인의 1:1 봇 채팅. 서버는 사용자 데이터를 영구 저장하지 않습니다 — 봇이 본인의 텔레그램 채팅에 JSON 파일을 보내 핀(고정)할 뿐, 데이터의 영구 보관은 텔레그램 클라우드가 담당합니다.
  • cred = chat_id × HMAC. 자격 증명은 본인의 chat_id 와 봇 토큰 기반 비밀로 만든 HMAC 서명입니다. 봇 토큰을 모르면 가짜 cred 위조가 불가능합니다.
  • 다른 사용자는 본인의 데이터에 접근 불가. 봇은 사용자 각각의 1:1 채팅에만 메시지를 쓰며, A 의 채팅에서 B 의 채팅을 볼 방법이 없습니다.
  • 4자리 확인 단계. 어깨너머 도용 차단을 위해, 텔레그램에서 봇 시작 → 봇이 보내준 4자리 코드를 앱에 입력해야 페어링이 완료됩니다. 정상 사용자만 본인의 텔레그램 알림으로 그 4자리를 받을 수 있습니다.
④ 데이터 폐기 절차
  • 브라우저 데이터: DevTools → Application → Local Storage → seed:* 키 전부 삭제. 또는 사이트별 데이터 지우기.
  • 텔레그램 백업: 봇 채팅 열고 핀된 JSON 메시지 삭제. 봇을 차단하면 더 이상 메시지가 오지 않음.
  • 운영자 측 별도 절차 불필요. 애초에 사용자별 데이터를 보관하지 않으므로 삭제 요청을 받을 대상이 없습니다.

오픈소스 — 동작은 GitHub 에서 직접 확인하실 수 있습니다. 서버 코드(server.js) 의 모든 엔드포인트 동작이 그대로 공개되어 있습니다.

기기 간 동기화

이 인스턴스에는 텔레그램 봇이 설정되지 않았습니다. 운영자가 TELEGRAM_BOT_TOKEN 환경 변수를 추가하면 다른 PC·모바일 간 동기화가 가능해집니다.

텔레그램 계정을 한 번 연동해두면 모든 자산·거래·스냅샷이 본인의 텔레그램 채팅 에 백업됩니다. 다른 PC 에서 같은 텔레그램으로 페어링하시면 그대로 복원됩니다.

  • 저장소 = 본인의 채팅. 서버는 데이터를 영구 보관하지 않습니다 — 봇이 JSON 파일을 본인의 채팅에 보내고 핀(고정)합니다
  • 다른 사용자가 접근 불가. 자격 증명은 본인의 chat_id 와 봇 토큰 기반 HMAC 서명. 봇은 본인의 1:1 채팅에만 메시지 작성 가능
  • 2단계 페어링. 6자리 코드 → 텔레그램에서 봇 시작 → 봇이 보낸 4자리 확인 코드를 앱에 입력. 어깨너머 도용 차단
  • 해제는 언제든 가능하며, 텔레그램에서 봇 차단으로도 가능합니다
